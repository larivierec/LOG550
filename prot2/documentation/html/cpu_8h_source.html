222 /*
223 *********************************************************************************************************
224 * CRITICAL SECTION CONFIGURATION
225 *
226 * Note(s) : (1) Configure CPU_CFG_CRITICAL_METHOD with CPU's/compiler's critical section method :
227 *
228 * Enter/Exit critical sections by ...
229 *
230 * CPU_CRITICAL_METHOD_INT_DIS_EN Disable/Enable interrupts
231 * CPU_CRITICAL_METHOD_STATUS_STK Push/Pop interrupt status onto stack
232 * CPU_CRITICAL_METHOD_STATUS_LOCAL Save/Restore interrupt status to local variable
233 *
234 * (a) CPU_CRITICAL_METHOD_INT_DIS_EN is NOT a preferred method since it does NOT support
235 * multiple levels of interrupts. However, with some CPUs/compilers, this is the only
236 * available method.
237 *
238 * (b) CPU_CRITICAL_METHOD_STATUS_STK is one preferred method since it supports multiple
239 * levels of interrupts. However, this method assumes that the compiler provides C-level
240 * &/or assembly-level functionality for the following :
241 *
242 * ENTER CRITICAL SECTION :
243 * (1) Push/save interrupt status onto a local stack
244 * (2) Disable interrupts
245 *
246 * EXIT CRITICAL SECTION :
247 * (3) Pop/restore interrupt status from a local stack
248 *
249 * (c) CPU_CRITICAL_METHOD_STATUS_LOCAL is one preferred method since it supports multiple
250 * levels of interrupts. However, this method assumes that the compiler provides C-level
251 * &/or assembly-level functionality for the following :
252 *
253 * ENTER CRITICAL SECTION :
254 * (1) Save interrupt status into a local variable
255 * (2) Disable interrupts
256 *
257 * EXIT CRITICAL SECTION :
258 * (3) Restore interrupt status from a local variable
259 *
260 * (2) Critical section macro's most likely require inline assembly. If the compiler does NOT
261 * allow inline assembly in C source files, critical section macro's MUST call an assembly
262 * subroutine defined in a 'cpu_a.asm' file located in the following software directory :
263 *
264 * <CPU-Compiler Directory><cpu><compiler>\
265 *
266 * where
267 * <CPU-Compiler Directory> directory path for common CPU-compiler software
268 * <cpu> directory name for specific CPU
269 * <compiler> directory name for specific compiler
270 *
271 * (3) (a) To save/restore interrupt status, a local variable 'cpu_sr' of type 'CPU_SR' MAY need
272 * to be declared (e.g. if 'CPU_CRITICAL_METHOD_STATUS_LOCAL' method is configured).
273 *
274 * (1) 'cpu_sr' local variable SHOULD be declared via the CPU_SR_ALLOC() macro which, if
275 * used, MUST be declared following ALL other local variables.
276 *
277 * Example :
278 *
279 * void Fnct (void)
280 * {
281 * CPU_INT08U val_08;
282 * CPU_INT16U val_16;
283 * CPU_INT32U val_32;
284 * CPU_SR_ALLOC(); MUST be declared after ALL other local variables
285 * :
286 * :
287 * }
288 *
289 * (b) Configure 'CPU_SR' data type with the appropriate-sized CPU data type large enough to
290 * completely store the CPU's/compiler's status word.
291 *********************************************************************************************************
292 */
293 /*$PAGE*/
