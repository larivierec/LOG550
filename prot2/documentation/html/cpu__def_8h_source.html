57 * x denotes major software version revision number
58 * yy denotes minor software version revision number
59 * zz denotes sub-minor software version revision number

61 * (b) The software version label #define is formatted as follows :
62 *
63 * ver = x.yyzz * 100 * 100
64 *
65 * where
66 * ver denotes software version number scaled as an integer value
67 * x.yyzz denotes software version number, where the unscaled integer
68 * portion denotes the major version number & the unscaled
69 * fractional portion denotes the (concatenated) minor
70 * version numbers
71 *********************************************************************************************************

131 /*
132 *********************************************************************************************************
133 * CRITICAL SECTION CONFIGURATION
134 *
135 * Note(s) : (1) Configure CPU_CFG_CRITICAL_METHOD with CPU's/compiler's critical section method :
136 *
137 * Enter/Exit critical sections by ...
138 *
139 * CPU_CRITICAL_METHOD_INT_DIS_EN Disable/Enable interrupts
140 * CPU_CRITICAL_METHOD_STATUS_STK Push/Pop interrupt status onto stack
141 * CPU_CRITICAL_METHOD_STATUS_LOCAL Save/Restore interrupt status to local variable
142 *
143 * (a) CPU_CRITICAL_METHOD_INT_DIS_EN is NOT a preferred method since it does NOT support
144 * multiple levels of interrupts. However, with some CPUs/compilers, this is the only
145 * available method.
146 *
147 * (b) CPU_CRITICAL_METHOD_STATUS_STK is one preferred method since it supports multiple
148 * levels of interrupts. However, this method assumes that the compiler provides C-level
149 * &/or assembly-level functionality for the following :
150 *
151 * ENTER CRITICAL SECTION :
152 * (1) Push/save interrupt status onto a local stack
153 * (2) Disable interrupts
154 *
155 * EXIT CRITICAL SECTION :
156 * (3) Pop/restore interrupt status from a local stack
157 *
158 * (c) CPU_CRITICAL_METHOD_STATUS_LOCAL is one preferred method since it supports multiple
159 * levels of interrupts. However, this method assumes that the compiler provides C-level
160 * &/or assembly-level functionality for the following :
161 *
162 * ENTER CRITICAL SECTION :
163 * (1) Save interrupt status into a local variable
164 * (2) Disable interrupts
165 *
166 * EXIT CRITICAL SECTION :
167 * (3) Restore interrupt status from a local variable
168 *
169 * (2) Critical section macro's most likely require inline assembly. If the compiler does NOT
170 * allow inline assembly in C source files, critical section macro's MUST call an assembly
171 * subroutine defined in a 'cpu_a.asm' file located in the following software directory :
172 *
173 * <CPU-Compiler Directory><cpu><compiler>\
174 *
175 * where
176 * <CPU-Compiler Directory> directory path for common CPU-compiler software
177 * <cpu> directory name for specific CPU
178 * <compiler> directory name for specific compiler
179 *
180 * (3) (a) To save/restore interrupt status, a local variable 'cpu_sr' of type 'CPU_SR' MAY need
181 * to be declared (e.g. if 'CPU_CRITICAL_METHOD_STATUS_LOCAL' method is configured).
182 *
183 * (1) 'cpu_sr' local variable SHOULD be declared via the CPU_SR_ALLOC() macro which,
184 * if used, MUST be declared following ALL other local variables (see any 'cpu.h
185 * CRITICAL SECTION CONFIGURATION Note #3a1').
186 *
187 * Example :
188 *
189 * void Fnct (void)
190 * {
191 * CPU_INT08U val_08;
192 * CPU_INT16U val_16;
193 * CPU_INT32U val_32;
194 * CPU_SR_ALLOC(); MUST be declared after ALL other local variables
195 * :
196 * :
197 * }
198 *
199 * (b) Configure 'CPU_SR' data type with the appropriate-sized CPU data type large enough to
200 * completely store the CPU's/compiler's status word.
201 *********************************************************************************************************
202 */
203

207 #define CPU_CRITICAL_METHOD_STATUS_STK 2u /* Push/Pop int status onto stk (see Note #1b). */
208 #define CPU_CRITICAL_METHOD_STATUS_LOCAL 3u /* Save/Restore int status to local var (see Note #1c). */
